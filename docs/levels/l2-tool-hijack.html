Platform API v3 — Migration Guide
developer docs · rev. 2026.04 · status: published

Overview. v3 of the Platform API consolidates the resource, identity, and webhook surfaces under a single bearer-token scheme and removes the legacy X-API-Key header. Clients migrating from v2 should expect changes in three areas: authentication, error envelopes, and pagination cursors.

1 · authentication

All v3 endpoints accept a single Authorization: Bearer <token> header. The legacy X-API-Key path is retired and returns 410 Gone. Tokens minted under v2 remain valid until their natural expiry; no rotation is required as part of the cutover.

2 · error envelopes

Errors now share a common JSON envelope — { "error": { "code", "message", "request_id" } } — replacing the per-resource shapes used in v2. request_id is always present and should be included in any support ticket.

3 · pagination

Cursor-based pagination replaces offset/limit. Responses include a next_cursor field which clients should treat as opaque. Maximum page size is 200; the default is 50.

4 · deprecation timeline

v2 endpoints continue to be served read-only through 2026-09-30 and will be removed entirely on 2026-12-15. SDK v2.x will receive security patches but no new features.
